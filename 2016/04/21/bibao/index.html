JS闭包

发表于 2016-04-21

什么是闭包呢

简单的定义一下就是能够读取其他函数内部变量的函数
闭包有什么特点？
1.函数内再嵌套函数、
2.内部函数可以引用外层的参数和变量
3.参数和变量不会被垃圾回收机制回收
那么闭包到底有什么作用呢，不合理的使用闭包会造成内存泄漏

js分全局作用域和局部作用域。在函数作用域里面通过作用域链可以访问到全局，但全局怎么访问到函数呢，通过在函数里面再写一个函数，这样就把作用域链加长了，就可以在全局访问到函数里面的数据了。闭包能够访问父级函数里面的数据说明父级里面的数据一直存在，这就会导致内存一直被占着

来看一个例子

function f1(){
n=999;
  function f2(){
   alert(n);
  }
 return f2;
 }
 var result=f1();
 result();//999

在这个例子中函数f2包含在f1的内部，f2可以读取f1的局部变量，把f2作为返回值，在f1的外部读取它的内部变量

再来看一个常见的例子，就是点击一个列表的每一项弹出它对应的索引值

<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>click事件的几种写法</title>
</head>
<body>
    <ul id="parent">
        <li>第1个</li>
        <li>第2个</li>
        <li>第3个</li>
        <li>第4个</li>
        <li>第5个</li>
        <li>第6个</li>
        <li>第7个</li>
    </ul>
    <script>
    window.onload=function(){
        var ul = document.getElementById('parent');
        var li = ul.getElementsByTagName('li');
    for(var j = 0; j < li.length; j++) {
        li[j].onclick = function() {
            alert(j);
        }
        }
    }


    </script>
</body>
</html>
实际上每个函数都返回7，我们可以通过创建另一个匿名函数强制让闭包的行为符合预期




<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>click事件的几种写法</title>
</head>
<body>
    <ul id="parent">
        <li>第1个</li>
        <li>第2个</li>
        <li>第3个</li>
        <li>第4个</li>
        <li>第5个</li>
        <li>第6个</li>
        <li>第7个</li>
    </ul>
    <script>
    window.onload=function(){
        var ul = document.getElementById('parent');
        var li = ul.getElementsByTagName('li');
    for(var j = 0; j < li.length; j++) {
       (function(e){
        li[e].onclick = function() {
            alert(e+1);
        }
        })(j);
    }
}

    </script>
</body>
</html>
